## Design Decisions (8)
1. Использование FastAPI - быстрое прототипирование.
2. SQLite в качестве хранилища - упрощённая настройка и переносимость.
3. SQLModel - упрощённое ORM + синхронизация с FastAPI.
4. Логирование запросов - middleware для перехвата запросов и записей логов.
5. Структура проекта - разделение на models, api, services, tests для SoC.
6. /user endpoints реализуют все CRUD операции.
7. offset/limit - пагинация для GET /user/.
8. Создание дополнительного GET /user/search - поиск пользователей по логину.

## Документация по API

### Endpoints

- GET /user/  
  - Параметры: offset, limit  
  - Ответ: Список пользователей

- GET /user/{user_id}  
  - Ответ: Объект пользователя

- GET /user/search?login=... (новый endpoint)
  - Описание: Поиск пользователей по строке login
  - Ответ: Список найденных пользователей

- POST /user/
  - Тело: { "login": str, "password": str }
  - Ответ: Созданный объект пользователя

- PUT /user/{user_id}
  - Тело: Любые поля для обновления (login, password)
  - Ответ: Обновленный объект пользователя

- DELETE /user/{user_id}
  - Ответ: { "ok": True }

## Тестирование API с Postman

Для каждого Endpoint не менее двух тестов (пример):  
1. Позитивный сценарий (корректные данные).  
2. Негативный сценарий (ошибочные данные).

Пример:  
- GET /user/  
  - Тест 1: Без параметров offset, limit → полная выдача  
  - Тест 2: offset=5, limit=2 → проверка пагинации
